操作系统Winsock错误

如果出现以下情况，可能是由于操作系统Winsock错误导致，请进行Winsock修复：

1、服务器突然断电或死机重启后，OA各项服务启动正常，但无法访问。

2、输入用户名和密码之后系统会停留在登陆界面,如“正在登陆中，请稍后。。。”

3、服务无法启动，服务卸载、服务注册无效，事件查看器中也无报错信息。

4、服务启动后停止。

5、Office_Task服务正常启动，在用户管理模块编辑用户时提示“目标机器积极拒绝”或“由于连接方在一段时间后没有正确答复或者连接的主机没有反应，连接尝试失败”。

6、MYOA\logs\error.log记录大量的错误日志“错误:[error] (OS 10038)在一个非套接字上尝试了一个操作”。

7、局域网访问正常，但不能连接外网。

8、可以登录QQ等，但无法打开网页。

9、局域网、外网全部无法连通，用Ping命令对网络进行检测的时候，PC喇叭会短鸣一声，并提示“Destination host unreachable”。

修复步骤为：打开“开始菜单 -> 运行”，输入“cmd”，在打开的命令行窗口中输入“netsh winsock reset”命令后回车，然后重启计算机。
